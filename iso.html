1. Introducción al Modelo ISO/OSI

El Modelo OSI (Open Systems Interconnection) es un marco de referencia desarrollado por la Organización Internacional de Normalización (ISO) en 1984. Este modelo divide las funciones de comunicación de red en siete capas abstractas, cada una con responsabilidades específicas.

El propósito principal del modelo OSI es facilitar la interoperabilidad entre diferentes sistemas y tecnologías de red mediante la estandarización de protocolos y funciones.

Las siete capas del modelo OSI son:

  1. Capa Física
  2. Capa de Enlace de Datos
  3. Capa de Red
  4. Capa de Transporte
  5. Capa de Sesión
  6. Capa de Presentación
  7. Capa de Aplicación

2. Capa Física

1 Capa Física

La capa física es la más baja del modelo OSI y se encarga de la transmisión de bits a través del medio de comunicación. Define las características eléctricas, mecánicas, funcionales y procedimentales para activar, mantener y desactivar el enlace físico entre sistemas.

Funciones principales:

  • Definición de medios físicos (cables, conectores)
  • Especificación de señales eléctricas u ópticas
  • Transmisión de bits crudos a través del medio
  • Control de la velocidad de transmisión
  • Especificación de topologías de red físicas

Ejemplos: Cableado Ethernet, cables de fibra óptica, señales Wi-Fi, estándares RS-232.

3. Capa de Enlace de Datos

2 Capa de Enlace de Datos

Esta capa proporciona transferencia confiable de datos a través del enlace físico. Se encarga de la detección y corrección de errores que puedan ocurrir en la capa física, así como del control de flujo entre dispositivos conectados directamente.

Funciones principales:

  • División de datos en tramas
  • Direccionamiento físico (direcciones MAC)
  • Detección y corrección de errores
  • Control de acceso al medio (MAC)
  • Control de flujo entre dispositivos adyacentes

Ejemplos: Protocolos Ethernet, PPP (Point-to-Point Protocol), switches de capa 2.

4. Capa de Red

3 Capa de Red

La capa de red se encarga del direccionamiento lógico y el enrutamiento de paquetes a través de múltiples redes. Su función principal es determinar la ruta óptima para enviar datos desde el origen al destino, incluso cuando están en redes diferentes.

Funciones principales:

  • Direccionamiento lógico (direcciones IP)
  • Enrutamiento de paquetes entre redes
  • Fragmentación de paquetes cuando es necesario
  • Control de congestión de red
  • Interconexión de diferentes tipos de redes

Ejemplos: Protocolo IP (Internet Protocol), routers, protocolos de enrutamiento como OSPF y BGP.

5. Capa de Transporte

4 Capa de Transporte

Esta capa proporciona transferencia confiable de datos entre los sistemas finales (hosts). Se encarga de la segmentación de datos, control de flujo extremo a extremo, detección y corrección de errores, y multiplexación de conexiones.

Funciones principales:

  • Segmentación y reensamblaje de datos
  • Control de flujo extremo a extremo
  • Control de errores y recuperación
  • Multiplexación de conexiones
  • Establecimiento, mantenimiento y liberación de conexiones

Ejemplos: Protocolos TCP (Transmission Control Protocol) y UDP (User Datagram Protocol).

6. Capa de Sesión

5 Capa de Sesión

La capa de sesión establece, gestiona y finaliza las conexiones (sesiones) entre aplicaciones. Controla el diálogo entre computadoras, permitiendo la comunicación bidireccional simultánea o alternada, y gestiona puntos de control y recuperación.

Funciones principales:

  • Establecimiento, mantenimiento y terminación de sesiones
  • Coordinación del diálogo entre aplicaciones
  • Sincronización de puntos de control
  • Control de tokens para acceso compartido
  • Recuperación de sesiones interrumpidas

Ejemplos: Protocolos RPC (Remote Procedure Call), NetBIOS, sesiones SSL/TLS.

7. Capa de Presentación

6 Capa de Presentación

Esta capa se encarga de la representación de la información, asegurando que los datos enviados por la capa de aplicación de un sistema puedan ser interpretados por la capa de aplicación de otro sistema. Maneja la traducción, compresión y cifrado de datos.

Funciones principales:

  • Traducción entre diferentes formatos de datos
  • Compresión y descompresión de datos
  • Cifrado y descifrado de información
  • Conversión de codificaciones de caracteres
  • Presentación uniforme de datos independiente del sistema

Ejemplos: Formatos JPEG, MPEG, ASCII a EBCDIC, algoritmos de compresión, cifrado SSL/TLS.

8. Capa de Aplicación

7 Capa de Aplicación

Es la capa más cercana al usuario y proporciona interfaces y servicios para que las aplicaciones puedan acceder a la red. No se refiere a las aplicaciones en sí, sino a los servicios que estas utilizan para comunicarse a través de la red.

Funciones principales:

  • Identificación de socios de comunicación
  • Determinación de la disponibilidad de recursos
  • Sincronización de comunicación
  • Transferencia de información
  • Provisión de servicios de red a aplicaciones

Ejemplos: Protocolos HTTP, FTP, SMTP, DNS, Telnet.

9. Resumen y Comparación

El modelo OSI proporciona una estructura conceptual que permite comprender cómo interactúan las diferentes funciones de red. Aunque el modelo TCP/IP es más utilizado en la práctica, el modelo OSI sigue siendo una referencia importante para la enseñanza y el diseño de redes.

Capa Función Principal Unidad de Datos Ejemplos
Aplicación (7) Interfaz entre aplicaciones y servicios de red Datos HTTP, FTP, SMTP
Presentación (6) Traducción, cifrado y compresión de datos Datos JPEG, SSL, ASCII
Sesión (5) Establecimiento y control de sesiones Datos NetBIOS, RPC
Transporte (4) Transferencia confiable extremo a extremo Segmentos/Datagramas TCP, UDP
Red (3) Direccionamiento lógico y enrutamiento Paquetes IP, routers
Enlace (2) Transferencia confiable entre nodos adyacentes Tramas Ethernet, switches
Física (1) Transmisión de bits a través del medio Bits Cables, señales, conectores

Es importante recordar que en la práctica, muchas implementaciones de red combinan funciones de varias capas del modelo OSI, especialmente en el modelo TCP/IP que tiene solo cuatro capas.
